# Shared Learnings — Система обучения конструктора

> Конструктор умнеет с каждым постом. Этот скилл собирает, хранит и применяет выученные правила.

---

## Быстрый старт

| Команда | Что делает |
|---------|------------|
| `что сработало` | Записать успешный паттерн |
| `что не сработало` | Записать антипаттерн |
| `покажи правила` | Показать текущие правила для контента |
| `анализ результатов` | Разобрать метрики последних постов → извлечь паттерны |
| `чеклист публикации` | Проверить пост перед публикацией |

---

## Как устроено обучение

### 3 слоя знаний

```
1. УНИВЕРСАЛЬНЫЕ правила (references/)
   → Работают для всех. Не меняются без причины.
   → Источник: проверенные методологии

2. ПРОЕКТНЫЕ правила ({project}/learning/)
   → Работают для конкретного эксперта/проекта.
   → Источник: фидбэк от audience-lens + метрики

3. СЕССИОННЫЕ правила (в памяти)
   → Работают прямо сейчас. Гипотезы.
   → Источник: текущая сессия работы
```

### Цикл обучения

```
Создать контент (text-posts, reels, carousel...)
    ↓
Проверить через audience-lens
    ↓
Опубликовать
    ↓
Собрать метрики (24-48 часов)
    ↓
Классифицировать: HIT / OK / MISS
    ↓
Извлечь паттерн / антипаттерн
    ↓
Записать в {project}/learning/
    ↓
Применить в следующем контенте
    ↻
```

## Запись паттернов

### Формат паттерна (что работает)

```markdown
## [Дата] Паттерн: [название]

**Контекст:** [где применялся — платформа, формат]
**Суть:** [1-2 предложения — что именно сделали]
**Результат:** [метрики или качественная оценка]
**Подтверждений:** [сколько раз сработало — минимум 3 для "правила"]
**Правило:** [готовая инструкция для будущего контента]
**Пример:**
> [конкретный текст/хук/структура]
```

### Формат антипаттерна (что НЕ работает)

```markdown
## [Дата] Антипаттерн: [название]

**Контекст:** [где провалилось]
**Суть:** [что сделали не так]
**Результат:** [метрики или качественная оценка]
**Почему не работает:** [причина]
**Правило:** [что делать вместо этого]
**Пример:**
> [конкретный текст, который провалился]
```

### Правило "3 подтверждения"

- **1 раз** = наблюдение (записать, но не менять правила)
- **2 раза** = гипотеза (тестировать осознанно)
- **3+ раз** = правило (добавить в approved/anti-patterns)

## Чеклист публикации

Перед каждой публикацией проверь:

### Обязательные проверки

- [ ] Хук первой строки — цепляет за 2 секунды?
- [ ] Нет AI-маркеров (канцелярит, "бесшовный", "революционный")
- [ ] Текст звучит как автор (проверить по voice-style.md)
- [ ] Конкретика вместо абстракций (цифры, примеры, истории)
- [ ] CTA — понятно что делать дальше?
- [ ] Длина соответствует платформе

### Антиспам-проверки

- [ ] Нет слов из стоп-листа (references/UNIVERSAL-ANTI-PATTERNS.md)
- [ ] Нет шаблонных переходов ("Более того", "Стоит отметить", "Давайте разберёмся")
- [ ] Нет фальшивой эмпатии ("Мы все знаем как это...")
- [ ] Нет "водянистых" списков (5 пунктов, говорящих одно и то же)

### Стратегические проверки

- [ ] Пост усиливает позиционирование? (не размывает бренд)
- [ ] Пост для конкретного сегмента ЦА? (не "для всех")
- [ ] Пост ведёт к следующему шагу в воронке?

## Связки

| Откуда | Что получаем |
|--------|-------------|
| `audience-lens` | Фидбэк HIT/OK/MISS → паттерны |
| `swipefile` | Разборы конкурентов → чужие приёмы |
| `tone-of-voice` | Калибровки голоса → обновление тон-формул |

| Куда | Что передаём |
|------|-------------|
| ВСЕ контент-скиллы | Правила и чеклист → качество контента |
| `audience-lens` | Антиспам-правила → экспертная линза |
| `tone-of-voice` | Тон-формулы → калибровка голоса |

---

## Когда загружать этот скилл

- **Всегда** при создании контента — чеклист + правила
- **После публикации** — запись фидбэка
- **Каждый месяц** — ревизия правил (что устарело?)
- **При смене стратегии** — пересмотр всех правил

---

## Антипаттерны обучения

| Ошибка | Почему плохо |
|--------|-------------|
| Не записывать результаты | Система не учится, повторяет ошибки |
| Делать правило после 1 случая | Случайность, не паттерн |
| Не пересматривать правила | То что работало полгода назад — может не работать |
| Копить примеры без анализа | Папка examples растёт, но инсайтов нет |
| Игнорировать MISS-ы | Из провалов учишься больше чем из успехов |
