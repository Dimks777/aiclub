# Universal Anti-Patterns

> Что НЕ работает — собрано из всех проектов.
> Новые проекты наследуют эти паттерны с первого дня.

**Источники:** expert-threads (100+ примеров), expert-carousel (3), expert-video (3)
**Последнее обновление:** 2026-01-24

## КРИТИЧЕСКИЕ (исправлять всегда)

### 1. "AI" вместо "ИИ" в русском тексте

**Частота:** 100+ нарушений (expert-threads)

**Проблема:**
Claude генерирует "AI", пользователь исправляет на "ИИ" каждый раз.

**Примеры:**
- ❌ "AI помог мне" → ✅ "ИИ помог мне"
- ❌ "спрашивали AI" → ✅ "спрашивали ИИ"
- ❌ "AI стал моим зеркалом" → ✅ "ИИ стал моим зеркалом"

**Исключения (НЕ заменять):**
- ChatGPT, OpenAI, Claude (названия продуктов/компаний)

**Checklist:**
```
ПЕРЕД ПУБЛИКАЦИЕЙ: Ctrl+F "AI" → заменить на "ИИ"
```

---

### 2. Стрелки "→" в тексте

**Частота:** 50+ нарушений (expert-threads)

**Проблема:**
Claude добавляет стрелки для красоты, пользователь удаляет.

**Примеры:**
- ❌ "Больше инсайтов → https://t.me/{YOUR_CHANNEL}"
- ✅ "Больше инсайтов в https://t.me/{YOUR_CHANNEL}"
- ❌ "не инструментом → зеркалом"
- ✅ "не инструментом. Зеркалом."

**Замены:**
- → "в" (Больше инсайтов в...)
- → "–" (тире, если нужна пауза)
- → "." (точка + новое предложение)

**Checklist:**
```
ПЕРЕД ПУБЛИКАЦИЕЙ: Ctrl+F "→" → удалить или заменить
```

---

### 3. Стаккато (рубленые фразы)

**Частота:** Встречается в нескольких проектах (expert-threads, expert-video)

**Проблема:**
Короткие обрывистые фразы звучат роботизированно. Особенно критично для AI-озвучки.

**Примеры:**
- ❌ "Худший овощ для простаты. Сырой. Не жареный. Сельдерей."
- ✅ "Один овощ вредит твоей простате больше всех. И это не тот что ты думаешь."

- ❌ "Потом. Где-то на 70%. Всё замедляется."
- ✅ "Потом на 70% всё останавливается."

**Решение:**
Связные предложения вместо рубленых фраз.

## СТИЛИСТИКА

### 4. Пассивный залог вместо первого лица

**Проблема:**
Пассивный залог отдаляет от читателя.

**Примеры:**
- ❌ "Появляется скука. Возникает мысль."
- ✅ "Теряю интерес. Я ловлю себя на том, что..."
- ❌ "Работа идёт, люди приходят"
- ✅ "Работаю, ко мне приходят люди"

**Правило:**
ВСЕГДА первое лицо: "я чувствую", "я делаю", "я понял".

---

### 5. Паттерн ChatGPT "Это не X. Это Y."

**Проблема:**
Типичный паттерн GPT, звучит как нейросеть.

**Примеры:**
- ❌ "Это не инструмент. Это зеркало."
- ✅ "ИИ стал моим зеркалом. Не инструментом. Зеркалом."

**Решение:**
Разбивать на отдельные предложения, добавлять контекст.

---

### 6. Раннее раскрытие ответа в хуке

**Частота:** Встречается в video/reels контенте (expert-video)

**Проблема:**
Зритель получает ответ и уходит.

**Примеры:**
- ❌ "Помидоры бесполезны для простаты. Сырые помидоры."
- ✅ "Ты ешь один продукт для простаты каждую неделю. Но организм не получает от него ничего."

**Правило:**
Ответ раскрывать после 10 сек, не в хуке.

---

### 7. Менторский тон сверху

**Проблема:**
Мы на равных с читателем, не учитель-ученик.

**Примеры:**
- ❌ "Запомни это правило:"
- ✅ "Попробуй для себя:"
- ❌ "Ты должен сделать:"
- ✅ "Вот что я попробовал:"

**Решение:**
Тон приглашающий, как друг делится опытом.

## СТРУКТУРА

### 8. Посты длиннее лимита платформы

**Проблема:**
Threads лимит 500 символов, но для удобства чтения лучше 400.

**Решение:**
- Разбивать на thread (2-3 поста вместо 5)
- Использовать "Дальше 👇" между постами
- Claude может самостоятельно разбивать в thread формат

---

### 9. CTA смешан с контентом

**Проблема:**
CTA должен быть отдельным блоком, не в середине поста.

**Правильная структура:**
```
Основной контент (< 400 chars)

CTA блок:
Ещё про ИИ для жизни:
https://t.me/{YOUR_CHANNEL}
```

---

### 10. Нет CTA в посте

**Проблема:**
КАЖДЫЙ пост должен иметь ссылку на Telegram/подписку.

**Правило:**
- Single post → CTA в конце
- Thread → CTA в последнем посте (или в закрепе)

---

## CAROUSEL-СПЕЦИФИЧНЫЕ

### 11. Абстрактные claims про AI

**Проблема:**
"AI говорит за тебя" — слишком абстрактно.

**Решение:**
- Использовать конкретные action verbs
- Показывать ЧТО AI производит (не что он "является")
- Пример: "снимается на видео вместо тебя" ✓

---

### 12. Цена в caption карусели

**Проблема:**
Дублирует информацию (цена уже на слайдах).

**Решение:**
- Цену на visual слайды
- Caption = hook + value
- Избегать redundant information

---

### 13. "Листай" как CTA на карусели

**Проблема:**
Очевидно из формата карусели.

**Решение:**
- Пропускать очевидные CTAs
- Использовать пространство для value ("Подробности в закрепе")

---

## VIDEO-СПЕЦИФИЧНЫЕ

### 14. Имена пациентов (мультиязык)

**Проблема:**
Русское имя не работает в EN/ES/EE версиях.

**Примеры:**
- ❌ "Пациент Сергей. 54 года. Вставал 5 раз за ночь."
- ✅ "Каждый день миллионы мужчин встают пять раз за ночь."

## ТЕМЫ (project-specific, но частые)

### 15. Технические темы для lifestyle-аккаунта

**Если lifestyle-аккаунт:**
- ❌ Claude Code, n8n, workflows
- ❌ Контент-производство, фабрика
- ❌ API, автоматизация бизнеса
- ✅ ИИ для жизни (не для работы)
- ✅ Самопознание через GPT

### 16. Листиклы "5 способов" для личного аккаунта

**Проблема:**
Листиклы — формат для экспертного контента.

**Решение:**
Если Tips формат → личный опыт, не список фич.

## AI-СТОП-ЛИСТ (слова и фразы, выдающие генерацию)

### Слова-маркеры (убрать или заменить на конкретику)
```
уникальный -> что именно уникально? конкретика
инновационный / революционный / прорывной -> убрать
комплексный подход -> убрать (пустое)
эффективный / высокоэффективный -> показать цифрой
ключевой / важнейший / значимый -> доказать или убрать
выдающийся / безупречный / премиальный -> убрать
стремительный (рост) -> на сколько %?
всеобъемлющий / масштабируемый -> убрать
```

### Глаголы-маркеры (заменить на простые)
```
осуществлять -> делать
обеспечивать -> давать
являться -> быть (или убрать)
предоставлять -> давать
способствовать -> помогать
предусматривать -> включать
погрузиться -> разобраться / покопаться
оптимизировать -> улучшить / поправить
```

### Вводные-паразиты (убрать полностью)
```
Важно отметить, что...
Стоит подчеркнуть...
Следует учитывать...
Не секрет, что...
В современном мире...
На сегодняшний день...
В условиях (цифровизации/конкуренции)...
Давайте углубимся...
В рамках (проекта/подхода)...
```

### Конструкции-маркеры (переписать)
```
"Не просто X, а Y" -> переформулировать
"Не только X, но и Y" -> переформулировать
"Во-первых... Во-вторых..." -> убрать нумерацию
"С одной стороны... с другой..." -> живой пример
"Таким образом" / "В заключение" -> просто закончить
"Эксперты отмечают" -> кто конкретно?
```

### Деепричастные нагромождения
```
Максимум 1 деепричастие на предложение, лучше 0.
ДО: "Являясь ключевым фактором, обеспечивая коммуникацию..."
ПОСЛЕ: "Команда наладила связь. Всё прошло гладко."
```

### Структурные маркеры
```
Правило трёх ("скорость, качество, надёжность") -> разбить
Монотонные абзацы одной длины -> чередовать
Списки с жирными заголовками -> текст
Идеально параллельные конструкции -> ломать ритм
```

