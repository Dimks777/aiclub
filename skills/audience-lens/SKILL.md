# Audience Lens — Оценка контента глазами аудитории

> Каждый пост проходит через 5-10 живых аватаров ЦА + 3 универсальных критиков. Не угадывание — системная проверка перед публикацией.

## Workflow

### Шаг 0: Подготовка аватаров (один раз на проект)

Перед первым запуском нужны аватары. Два пути:

**Путь A — из customer-research:**
1. Открой `projects/{project}/CUSTOMER_PROFILE.md` (результат customer-research)
2. Возьми сегменты ЦА (обычно 3-5 штук)
3. Для каждого сегмента создай 1-2 аватара по шаблону из `references/avatar-template.md`
4. Итого: 5-10 аватаров
5. Сохрани в `projects/{project}/audience-avatars.md`

**Путь B — интервью с экспертом:**
1. Спроси эксперта: "Опиши 5-10 типичных людей из твоей аудитории"
2. Для каждого заполни шаблон аватара
3. Сохрани в `projects/{project}/audience-avatars.md`

**Минимум 5 аватаров. Оптимально 8-10.** Меньше 5 — слишком узкий срез, оценка ненадёжна.

### Шаг 1: Загрузка контекста

```
1. Прочитай projects/{project}/audience-avatars.md
2. Прочитай references/scoring-system.md
3. Получи контент для оценки от пользователя
```

### Шаг 2: Оценка аватарами (Уровень 1)

Для каждого аватара — войди в его роль полностью:
- Его жизненная ситуация, боли, желания
- Что его цепляет, что отталкивает
- Как он реагирует на контент (лайк, сохранение, пролистывание, отписка)

Каждый аватар оценивает контент по 4 метрикам (0-100):

| Метрика | Вопрос |
|---------|--------|
| **Relevance** | Это про меня? Про мою ситуацию? |
| **Emotional** | Цепляет эмоционально? Узнаю себя? |
| **Useful** | Могу применить? Дало что-то новое? |
| **Shareable** | Перешлю другу? Сохраню? Прокомментирую? |

**Формула:** `Total = Relevance×0.3 + Emotional×0.25 + Useful×0.25 + Shareable×0.2`

Каждый аватар также даёт:
- **Реакцию** (very_positive / positive / neutral / negative / skip)
- **Действие** (лайк, сохранение, комментарий, репост, отписка, пролистал)
- **Голос** — одну фразу от лица аватара ("Это прям про меня", "Ну и что?")
- **Критику** — что не зацепило или оттолкнуло

### Шаг 3: Оценка критиками (Уровень 2)

Три универсальных критика оценивают качество контента (1-100):

| Критик | Фокус | Ключевые вопросы |
|--------|-------|-----------------|
| **Sales** | Ведёт ли к действию? | CTA, триггеры, путь в воронку, следующий шаг |
| **Creative** | Цепляет ли? | Хук, уникальность, эмоция, свежесть подачи |
| **Structure** | Понятно ли? | Логика, ясность, читаемость, форматирование |

### Шаг 4: Вердикт

Комбинированное решение по порогам из `references/scoring-system.md`.

## Формат вывода

### Полная оценка

```markdown
## 👁️ Audience Lens: [название/первые слова поста]

### Уровень 1: Аватары ЦА

| Аватар | Score | Реакция | Голос |
|--------|-------|---------|-------|
| {Имя} ({возраст}) | XX% | 🔥/👍/😐/👎/⏭️ | "цитата" |
| ... | ... | ... | ... |

**Среднее аватаров:** XX%
**Лучший отклик:** {Имя} ({тема})
**Худший отклик:** {Имя} ({почему})

### Уровень 2: Критики

| Критик | Score | Ключевое |
|--------|-------|----------|
| Sales | XX/100 | [одно предложение] |
| Creative | XX/100 | [одно предложение] |
| Structure | XX/100 | [одно предложение] |

**Среднее критиков:** XX/100

### Вердикт

**Статус:** ✅ PUBLISH / ⚡ REVISE / ❌ REWRITE
**Тип контента:** Массовый / Нишевый (для {сегмент})
**Главная доработка:** [если нужна — одно конкретное действие]
**Сильная сторона:** [что работает лучше всего]
```

### Быстрая оценка (quick)

```markdown
## 👁️ Quick Lens: [название]

| Аватар | Score | Реакция |
|--------|-------|---------|
| {Имя} | XX% | 🔥/👍/😐/👎/⏭️ |
| ... | ... | ... |

**Среднее:** XX% → ✅/⚡/❌
```

## Что делать с результатом

| Вердикт | Действие |
|---------|----------|
| ✅ PUBLISH | Отправлять на публикацию |
| ✅ PUBLISH (нишевый) | Публиковать, пометить тегом сегмента |
| ⚡ REVISE | Передать копирайтеру с конкретной критикой аватаров |
| ❌ REWRITE | Переписать с нуля, учитывая что оттолкнуло аватаров |

---

## Анти-паттерны

- ❌ Запускать без аватаров — сначала `audience lens setup`
- ❌ Менее 5 аватаров — оценка будет перекошена
- ❌ Аватары без конкретной ситуации — "женщина 30 лет" не работает, нужна БОЛЬ и контекст
- ❌ Хвалить автоматически — аватары критичны, ищут слабые места
- ❌ Игнорировать нишевый контент — если один аватар ≥85%, это валидный пост для сегмента
- ❌ Использовать без customer-research — аватары из головы = оценка из головы

## Уровень 3: Экспертные линзы (опционально)

Помимо аватаров ЦА и универсальных критиков, можно подключить экспертные линзы — виртуальных экспертов по контенту, которые дают профессиональную оценку.

### Доступные линзы

| Линза | Фокус | Когда подключать |
|-------|-------|-----------------|
| **Копирайтер** | Текст: хуки, ритм, CTA, скользкая горка | Всегда для текстовых постов |
| **Стратег** | Позиционирование: это усиливает бренд или размывает? | Для серийного контента |
| **Маркетолог** | Воронка: куда ведёт пост? Какой следующий шаг? | Для продающего контента |
| **Антиспам** | AI-детект: звучит ли как AI? Есть ли слоп-маркеры? | Всегда |

### Формат оценки линзы

```
Линза: [Копирайтер]
Оценка: XX/100
Вердикт: [одно предложение]
Конкретная правка: [что именно изменить, с примером]
```

### Антиспам-линза (обязательная)

Проверяет текст на AI-маркеры. Красные флаги:

| Маркер | Пример | Замена |
|--------|--------|--------|
| Канцелярит | "В рамках данного" | "Вот что" |
| AI-слова | "бесшовный", "революционный" | конкретика |
| Пустые усилители | "невероятно важный" | убрать или заменить фактом |
| Шаблонные переходы | "Более того", "Стоит отметить" | убрать |
| Списки-заполнители | 5 пунктов сказавших одно и то же | сократить до 2-3 |
| Фальшивая эмпатия | "Мы все знаем как это..." | личная история |

## Фидбэк-луп: обучение на результатах

### После публикации (через 24-48 часов)

Собери метрики и классифицируй:

| Класс | Критерий | Действие |
|-------|----------|----------|
| **HIT** | Топ-20% по вовлечённости | Разобрать: что сработало? |
| **OK** | Средние показатели | Ничего не делать |
| **MISS** | Низ-20% по вовлечённости | Разобрать: что оттолкнуло? |

### Формат записи фидбэка

```markdown
## [Дата] [Формат] [Тема]
- Платформа: Threads / Telegram / Instagram
- Метрики: просмотры X, лайки Y, комментарии Z, сохранения W
- Класс: HIT / OK / MISS
- Audience Lens прогноз: XX% (совпал / не совпал)
- Что сработало: [конкретный элемент — хук, структура, CTA]
- Что не сработало: [конкретный элемент]
- Правило: [что повторять или избегать]
```

### Куда сохранять

```
Сохрани в: {project}/learning/content-feedback.md
```

### Правила обучения

- Паттерн становится правилом после 3+ подтверждений
- Пересмотр правил — раз в месяц
- HIT-ы сохранять как эталоны в `{project}/learning/examples/`
- MISS-ы анализировать: проблема в теме, подаче или платформе?

### Калибровка аватаров

Если Audience Lens систематически промахивается (прогноз ≠ реальность):

| Проблема | Решение |
|----------|---------|
| Аватары хвалят, аудитория молчит | Аватары слишком лояльны → ужесточить пороги |
| Аватары ругают, аудитория лайкает | Аватары устарели → обновить профили из свежих данных |
| Один аватар всегда 90%+ | Аватар слишком generic → конкретизировать боли |
| Все аватары одинаковые баллы | Аватары не различаются → пересоздать из разных сегментов |
