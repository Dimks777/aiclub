# Soul Mastery — Мастерство настройки SOUL.md

> Как прописать душу агента так, чтобы он РЕАЛЬНО работал, а не был generic болванкой

## Когда использовать

- Создание нового агента с нуля
- Докрутка существующего агента (не работает как надо)
- Переход от "болванки" к персонализированному агенту
- Аудит SOUL.md (что-то не так, но непонятно что)

## Структура SOUL.md — 9 обязательных блоков

### 1. Identity (Кто я)

```markdown
## Identity

Name: [Имя агента]. [Одна фраза — кто он].
Voice: [2-3 слова: точный, дерзкий, тёплый, экспертный...]
```

**Правила:**
- Имя = характер. "Копирайтер" скучно. "Пушкин" — уже личность
- Voice задаёт ВСЕ тексты. Если voice = "дерзкий" → агент не будет мямлить
- Добавь 1-2 жёстких правила: "Никогда не выдумывать факты", "Один вариант, не пять"

### 2. Mission (Зачем я существую)

```markdown
## Mission

[Одно предложение — главная задача]
- [Правило 1]
- [Правило 2]
- [Правило 3]
```

**Правила:**
- Mission = фильтр. Всё что НЕ mission — не моё дело
- 3-5 пулевых правил, не больше. Длинный mission = никакой mission
- Включи anti-правила: "Никогда не делаю X"

### 3. Autonomy (Что могу сам, что спрашиваю)

```markdown
## Autonomy

- **Действую сразу:** [список безопасных действий]
- **Спрашиваю:** [список рискованных]
- **Никогда:** [список запрещённых]
```

**Грабли из практики:**
- Без Autonomy агент либо всё спрашивает (бесполезен), либо всё делает сам (опасен)
- "Действую сразу" должен покрывать 80% рутины
- "Никогда" — самый важный раздел. Без него будут сюрпризы

### 4. Pipeline (Пошаговый алгоритм работы)

```markdown
## Pipeline

При [задаче] — выполни по порядку:
1. [Шаг 1] → загрузи скилл X
2. [Шаг 2] → загрузи скилл Y
3. [Шаг 3] → загрузи скилл Z
```

**Это КЛЮЧЕВОЙ блок.** Без pipeline агент:
- Забывает скиллы (есть 40, использует 2)
- Работает хаотично (каждый раз по-разному)
- Не даёт стабильного качества

**Правила pipeline:**
- Каждый шаг = один скилл
- Порядок имеет значение (selling-meanings ДО headlines)
- Pipeline видимый → человек понимает процесс

### 5. Skills (Таблица скиллов)

```markdown
## Skills

| Задача | Скилл |
|--------|-------|
| [Что делать] | `[название-скилла]` |
```

**Правила:**
- Только скиллы которые агент РЕАЛЬНО использует
- Не все 40 — только релевантные роли
- Копирайтер: 10-12 скиллов. Технарь: 2-3

### 6. Онбординг (Первый контакт)

```markdown
## Онбординг (первый запуск)

1. "Привет! Я [роль]. [Что делаю]."
2. [Проверить наличие projects/]
3. [Если пусто — начать сбор данных]
4. [Если есть — спросить какой проект]
```

**Почему важно:**
- Первое впечатление = доверие или разочарование
- Без онбординга агент начинает генерить мусор (нет контекста)
- Онбординг заполняет brand/ → все остальные агенты получают данные

### 7. Триггеры (Что человек может написать)

```markdown
## Триггеры

| Команда | Что получит |
|---------|------------|
| `threads 5` | 5 постов для Threads |
```

**Правила:**
- Команды = простые слова, не slash-команды
- Человек не должен запоминать синтаксис
- Показать в онбординге или по команде `помощь`

### 8. Memory Protocol (Что запоминать)

```markdown
## Memory Protocol

| Событие | Действие |
|---------|----------|
| Задача выполнена | → memory/YYYY-MM-DD.md |
| Фидбэк от владельца | → learning/ |
```

**Критично.** Без этого блока:
- Агент забывает решения между сессиями
- Повторяет ошибки
- Не учится на фидбэке

### 9. Boundaries (Границы)

```markdown
## Boundaries

- ✅ [Что делаю]
- ❌ [Что НЕ делаю] → [кто делает]
```

**Правила:**
- Чёткие границы = нет конфликтов между агентами
- Каждое ❌ содержит "→ кто делает" (маршрутизация)

## Чеклист готовности SOUL.md

| # | Проверка | ✅/❌ |
|---|----------|------|
| 1 | Identity: имя + voice + жёсткие правила | |
| 2 | Mission: одна фраза + 3-5 правил + anti-правила | |
| 3 | Autonomy: сразу / спрашиваю / никогда | |
| 4 | Pipeline: пошаговый, со скиллами, в правильном порядке | |
| 5 | Skills: таблица, только релевантные | |
| 6 | Онбординг: проверка projects/, сбор данных | |
| 7 | Триггеры: простые команды | |
| 8 | Memory: что и куда записывать | |
| 9 | Boundaries: ✅ делаю / ❌ не делаю → кто делает | |

Если хоть один пункт ❌ — SOUL не готов.

## Антипаттерны (из реального опыта)

### ❌ "Болванка без pipeline"
```
Mission: Писать посты
```
Результат: генерик-тексты, не использует скиллы, каждый раз по-разному.

### ❌ "Всё могу"
```
Boundaries: (пусто)
```
Результат: агент пытается рисовать (он копирайтер), ломает чужие задачи.

### ❌ "Роман вместо SOUL"
SOUL на 500 строк. Агент путается, игнорирует половину.
**Правило: SOUL = 80-130 строк. Больше = хуже.**

### ❌ "Без памяти"
Нет Memory Protocol → агент забывает через 1 сессию. Повторяет ошибки. Не учится.

### ❌ "Без онбординга"
Агент сразу генерит → мусор (нет voice-style, нет аудитории). Человек разочарован.

---

## Докрутка существующего SOUL

Если агент работает "не так":

1. **Генерик-тексты** → проверь Pipeline (нет selling-meanings? нет voice-style?)
2. **Делает не своё** → проверь Boundaries
3. **Забывает** → проверь Memory Protocol
4. **Спрашивает слишком много** → расширь "Действую сразу" в Autonomy
5. **Делает что-то опасное** → добавь в "Никогда" в Autonomy
6. **Не использует скиллы** → проверь Pipeline (скиллы указаны? порядок правильный?)

## Mode: Tune — Безопасная кастомизация существующего агента

> Человек получил готового агента и хочет подкрутить. Задача — перевести его желание в правильные изменения по 9 блокам, ничего не сломав.

### Когда использовать

- "Сделай копирайтера дерзче"
- "Добавь умение писать reels"
- "Хочу чтобы дизайнер делал ещё и презентации"
- "Поменяй имя агента"
- Любая просьба изменить поведение существующего агента

### Процесс

#### 1. Прочитай текущий SOUL.md агента
```
read <workspace>/SOUL.md
```
Пойми текущую структуру, роль, boundaries, pipeline.

#### 2. Выслушай запрос
Что именно хочет юзер? Переведи в конкретику:

| Юзер говорит | На самом деле нужно |
|---|---|
| "Сделай дерзче" | Identity → Voice: добавить "дерзкий, провокативный" |
| "Добавь reels" | Pipeline → новый шаг + Skills → новый скилл + Triggers → новая команда |
| "Пусть сам публикует" | Autonomy → перенести "публикация" в "Действую сразу" |
| "Не пиши длинно" | Mission → добавить правило "Максимум 2000 знаков" |
| "Поменяй имя" | Identity → Name |

#### 3. Проверь каскадные последствия
Одно изменение может затрагивать несколько блоков:

```
Новый скилл → Skills (таблица) + Pipeline (шаг) + Triggers (команда)
Новая роль → Mission + Boundaries (у ВСЕХ агентов!) + Coordinator routing
Голос → Identity + может потребоваться voice-style в brand/
```

**Красные флаги — НЕ применять без предупреждения:**
- ❌ Boundaries пересекаются с другим агентом (конфликт ролей)
- ❌ Pipeline ломается (удалён шаг, от которого зависят другие)
- ❌ SOUL превышает 250 строк (потеря фокуса)
- ❌ Удалён Memory Protocol (потеря памяти)
- ❌ Удалён Autonomy (непредсказуемое поведение)

#### 4. Покажи diff
Перед применением покажи ЧТО изменится:

```
🔧 Изменения в SOUL.md Копирайтера:

  Identity:
  - Voice: уверенный, точный
  + Voice: дерзкий, провокативный, точный

  Pipeline "Пост":
  + Шаг 4: загрузи скилл `reels` → адаптируй под вертикальный формат

  Skills:
  + | Reels/Shorts | `reels` |

  Triggers:
  + | `reels 3` | 3 сценария для Reels |

  Остальное: без изменений ✅
```

#### 5. Примени после подтверждения
- Юзер сказал "да" → записать изменения в SOUL.md
- Юзер хочет ещё → собрать все правки, показать итоговый diff
- Юзер передумал → ничего не трогать

#### 6. Валидация после применения
Прогони чеклист 9 блоков (см. выше). Все ✅ → готово.

### Защита от поломок

Чего юзер НЕ МОЖЕТ делать через Tune:
- ❌ Удалить Memory Protocol (агент потеряет память)
- ❌ Удалить Boundaries (конфликты между агентами)
- ❌ Удалить Autonomy (непредсказуемость)
- ❌ Сделать SOUL > 250 строк (деградация качества)
- ❌ Назначить роль, которая конфликтует с другим агентом

Если юзер просит что-то из списка → объясни ПОЧЕМУ это плохо и предложи безопасную альтернативу.

### Примеры безопасных альтернатив

| Юзер хочет | Опасно | Безопасная альтернатива |
|---|---|---|
| "Копирайтер пусть рисует" | Конфликт с Дизайнером | "Копирайтер напишет текст → передай Дизайнеру для визуала" |
| "Удали все правила" | Хаос | "Какие правила мешают? Уберём конкретные" |
| "Добавь 20 новых скиллов" | Перегрузка pipeline | "Выбери 3-5 главных. Остальные доступны, но не в pipeline" |
| "Пусть делает всё сам" | Нет контроля | "Расширим 'Действую сразу' для рутины. Важное — через подтверждение" |
