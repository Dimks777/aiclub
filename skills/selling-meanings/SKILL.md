# Selling Meanings — Продающие смыслы

> Мост между исследованием и контентом. Смыслы = топливо для прогрева, вебинара, лендинга, дожима.

---

## Быстрый старт

| Команда | Что делает |
|---------|------------|
| `/meanings extract [текст/транскрипт]` | Извлечь продающие смыслы из сырого материала |
| `/meanings map [продукт]` | Построить карту смыслов (20-30 штук) для продукта |
| `/meanings pack [смысл] [формат]` | Упаковать один смысл в конкретный формат (пост/reels/сторис/email) |

| `/meanings pain+case [боль]` | Создать связку "Боль + Кейс" — самую эффективную единицу контента |

---

## Что такое продающий смысл

Продающий смысл — это идея, которая:
1. **Резонирует** с болью/желанием аудитории
2. **Позиционирует** эксперта как решение
3. **Упаковывается** в любой формат (пост, рилс, сторис, лендинг, email)

Один сильный смысл > десять слабых. Смысл работает, когда человек читает и думает: *"Это про меня"*.

---

## Формула смысла

```
СМЫСЛ = Боль аудитории + Инсайт эксперта + Доказательство
```

### Пример

| Элемент | Содержание |
|---------|------------|
| **Боль** | "Не могу продавать, чувствую себя впаривателем" |
| **Инсайт** | "Продажа = помощь, если продукт реально решает проблему" |
| **Доказательство** | "Клиент пришёл со словами: 'наконец кто-то объяснил нормально'" |
| **Смысл** | Продавать — значит помогать найти решение, которое человек уже ищет |

---

## UMP/UMS — Уникальные механизмы

Самый мощный тип смысла — **UMP/UMS пара**: UMP объясняет НАСТОЯЩУЮ причину проблемы (неожиданно, с доказательствами), UMS показывает новый подход к решению, логически связанный с UMP.

> Полный фреймворк: `references/ump-ums.md`

## 5 типов смыслов

Для каждого продукта нужна карта из **всех типов**. Пропуск любого = дыра в прогреве.

| # | Тип смысла | Что закрывает | Пример |
|---|------------|---------------|--------|
| 1 | **Смыслы входа** | Почему начать сейчас | "Каждый месяц без системы = деньги мимо" |
| 2 | **Смыслы доверия** | Почему именно этот эксперт | "300+ учеников, которые уже внедрили" |
| 3 | **Смыслы результата** | Что получит клиент | "Через 2 недели — первые заявки" |
| 4 | **Смыслы срочности** | Почему не ждать | "Места ограничены / цена растёт / рынок меняется" |
| 5 | **Смыслы отстройки** | Чем отличается от конкурентов | "Не теория, а система с обратной связью" |
| 6 | **UMP-смыслы** | Почему старые способы не работают | "Проблема не в конверсии, а в продуктовой линейке" |
| 7 | **UMS-смыслы** | Новый подход к решению | "Множественные окна продаж вместо одного запуска" |

## Карта смыслов из кастдевов (метод Пырикова)

### Процесс создания

```
Распаковка эксперта + Анализ аудитории + Кастдевы
                        |
               КАРТА СМЫСЛОВ (20-30 штук)
                        |
      Стратегия прогрева -> Контент -> Вебинар -> Продажи
```

### Как строить карту

1. Провести 2-3 созвона на каждый сегмент ЦА
2. Выписать **ВСЕ** боли, желания, возражения (дословные цитаты!)
3. Добавить боли околоцелевой аудитории (расширяет воронку)
4. На каждую боль -> способ закрытия (кейс/продукт/опыт)
5. Результат: банк из 20-30 смыслов, рассортированных по 5 типам

### Формат записи каждого смысла

```
БОЛЬ: [конкретная формулировка из кастдева]
ЗАКРЫТИЕ: [как продукт/эксперт решает]
ДОКАЗАТЕЛЬСТВО: [кейс/цифра/факт]
ФОРМАТ: [сторис/пост/эфир/подкаст/reels/email]
```

### Правило: смыслы меняются

> Кастдевы перед КАЖДЫМ запуском. Тренды меняются, рынок меняется, появляются новые боли. Глаз замыливается. Старые смыслы теряют силу.

## Источники смыслов (приоритет)

| # | Источник | Ценность | Как извлекать |
|---|----------|----------|---------------|
| 1 | **Кастдевы** | Высшая | Обновлять перед каждым запуском — первичные данные |
| 2 | **Сообщения из Директа** | Высокая | Живые запросы и формулировки ЦА на их языке |
| 3 | **Анкеты предзаписи** | Высокая | Вопрос "ваш запрос на продукт" — готовые боли |
| 4 | **Эфиры и разборы** | Средняя | Вопросы участников = реальные проблемы |
| 5 | **Конкуренты** | Базовая | Что они НЕ говорят — зона отстройки |

Дополнительно: кейсы клиентов, отзывы с конкретикой, FAQ (скрытые боли), личный опыт эксперта.

## Связка "Боль + Кейс"

**Самая эффективная единица контента для прогрева.**

```
Утро: подняли боль/проблему/возражение
-> Вечер: закрыли кейсом

Повторять через день начиная со 2-й недели прогрева.
```

| Элемент | Что |
|---------|-----|
| **Боль** | Конкретная формулировка из кастдева (на языке клиента) |
| **Кейс** | Реальная история клиента с результатом |
| **Мост** | Как продукт/эксперт помог перейти из боли в результат |

Правила: кейсы под разные сегменты ЦА, цифры и конкретика > общие слова, живые цитаты > пересказ.

## Упаковка: один смысл -> множество форматов

| Формат | Как упаковать |
|--------|---------------|
| **Threads / пост** | Хук из смысла -> раскрытие в 500-1000 символов -> CTA |
| **Reels** | Визуальная метафора смысла, 15-60 сек, хук в первые 2 сек |
| **Сторис** | Опрос / вопрос / слайдер вокруг смысла |
| **Лендинг** | Блок с заголовком из смысла + раскрытие + доказательство |
| **Email** | Тема письма из смысла -> история -> CTA |
| **Вебинар** | Контентный блок + кейс-закрытие |

> **Правило:** один смысл можно и нужно использовать в 3-5 разных форматах. Повторение с разной упаковкой -- не надоедание.

## Выход: смысл -> контент

Каждый смысл из карты -- топливо для контента. Используй цепочку:

```
selling-meanings (карта смыслов)
        |
        +--> text-posts (структура поста + хук из смысла)
        |       |
        |       +--> threads (обертка: ветки, engagement)
        |       +--> telegram (обертка: форматирование, воронки)
        |
        +--> storytelling (если смысл = история)
        +--> reels (если смысл = визуал)
        +--> copywriting (если смысл -> лендинг/email)
```

### Быстрый путь: смысл -> пост

1. Выбери смысл из карты
2. Определи формат через **text-posts** (инструмент/закулисье/личное/философия/продукт)
3. Собери хук из боли (text-posts: hooks-and-cta.md)
4. Раскрой через структуру типа (text-posts: post-anatomy.md)
5. Адаптируй под платформу (threads/telegram)

## Антипаттерны и частые ошибки

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|-----------------|
| Смысл без конкретики | "Мы помогаем расти" — кому? как? на сколько? | Конкретные цифры и сроки |
| Смысл-клише | "Качество по доступной цене" — не цепляет | Уникальная формулировка на языке ЦА |
| Смысл без доказательства | Голословное утверждение — нет доверия | Кейс / цифра / цитата клиента |
| Один смысл на всё | Один пост — и всё? | Карта из 20-30 смыслов, 5 типов |
| Смыслы из головы | Продюсер придумал, не клиент сказал | Только из кастдевов и реальных данных |
| Смыслы без упаковки | Карта есть, контента нет | Каждый смысл -> 3-5 единиц контента |
| Одинаковые смыслы для всех сегментов | Мама в декрете и предприниматель — разные боли | Сегментировать карту по ЦА |
| Не обновлять карту | Старые смыслы теряют силу | Обновлять перед каждым запуском |

## Главный принцип

```
Кастдев -> Карта смыслов -> Контент -> Прогрев -> Продажи

Смыслы — это ТОПЛИВО. Без топлива двигатель не работает.
Один кастдев = 20-30 смыслов = контент на месяц.
```
