# UMP/UMS — Уникальные механизмы (Кеннеди + Скрижаль)

> Полный фреймворк для создания уникальных механизмов проблемы и решения.

---

## UMP/UMS пара

Самый мощный тип смысла — **UMP/UMS пара**:

```
UMP (Уникальный Механизм Проблемы):
  «Вот НАСТОЯЩАЯ причина, почему у вас не получается [результат]»
  → Удивляет, противоречит интуиции
  → Подкреплён доказательствами (исследования, цифры, имена)
  → Имеет запоминающийся псевдоним («Ловушка одного продукта»)

UMS (Уникальный Механизм Решения):
  «Вот что на самом деле нужно делать — и именно это мы делаем в [продукте]»
  → Логически связан с UMP (проблема = X → решение = анти-X)
  → Подводит к продукту, но ≠ продукт
```

---

## Как создать UMP/UMS для продукта

1. Определить главную боль ЦА (из кастдевов)
2. Найти НЕОЖИДАННУЮ причину этой боли (исследования, опыт, данные)
3. Дать причине запоминающееся название
4. Описать решение, логически закрывающее причину
5. Связать решение с продуктом

---

## Элементы сильного UMP

- **Конкретность**: имена + цифры = доверие
- **Проверяемое доказательство**: то, что человек может подтвердить на себе СРАЗУ
- **Противоречие интуиции**: реакция «серьёзно?!» ≠ банальность
- **Псевдоним**: запоминающееся название для механизма
